# NL-04 — Reddit / surveillance piece register

**Category:** NLG / generation  
**Added:** 2026-04-06  
**Non-negotiable:** No (Reddit-piece profile)

---

## Rule

Prose generated for writing-under-surveillance and Reddit-related pieces must maintain the register appropriate for a DHQ-adjacent argument about platform governance and student data. The piece addresses a specific population (CUNY students), a specific platform (Reddit), and a specific structural condition (platform data that persists beyond user deletion). All three must be named concretely throughout.

---

## Register characteristics

- Empirical and archival, not polemic
- Population named specifically: CUNY students, not "users" or "people"
- Platform behaviors named mechanically: not "Reddit surveils" but "Reddit's data retention pipeline preserves deleted content for 90 days"
- Data quantified where available; placeholders [X] used where pending
- Claims bounded to the dataset and time period

---

## The central argument structure

1. Reddit's platform mechanisms (ranking, retention, logging) shape which contributions persist and which disappear
2. CUNY students participate in campus subreddits without knowing this architecture
3. The mismatch between user expectations and platform behavior creates a structural vulnerability
4. This vulnerability is not addressed by current platform policies or university guidance
5. DH methods (corpus analysis, FOIA documentation) can make this architecture visible

Each generated section must be traceable to this structure.

---

## Vocabulary alignment

| Use | Avoid |
|---|---|
| data retention pipeline | surveillance apparatus |
| findable | surveilled |
| CUNY students | users / people |
| r/CUNY, r/Baruch | campus subreddits (unless making the general point) |
| compliance record | cooperation log |
| law enforcement requests | legal demands |
| [X] posts per day | significant activity / high volume |
