# AK-05 — No invented academic noun phrases

**Category:** Academic register  
**Added:** 2026-04-06  
**Non-negotiable:** Yes (cloze paper and Reddit-piece profiles)

---

## Rule

Don't coin noun phrases that sound scholarly but appear nowhere in the actual literature. If the field doesn't use the phrase, don't mint it. Use the terms researchers actually use in the relevant literature. If you're uncertain whether a phrase is established, treat it as invented until you can cite a source using it.

---

## The pattern to catch

LLMs generate plausible-sounding academic compounds: "LLM-derived predictability," "semantic retrieval interference," "distributional alignment," "occlusion-based inference," "reading trajectory asymmetry." These are not field terms. They sound like field terms. They're not.

---

## Diagnostic

For any noun phrase that could not appear in a paper title without sounding strange to researchers in the field: stop. Search. If no peer-reviewed source uses the phrase, don't use it. Use the term the field actually uses.

---

## Field terms for relevant domains

**Cloze procedure / reading research:**
cloze probability, cloze completion rate, reading time, eye-tracking, fixation duration, regression, first-pass reading time, surprisal, predictability, contextual constraint, lexical access, incremental processing, N-gram probability, language model surprisal

**Platform studies / digital humanities:**
content moderation, platform governance, algorithmic curation, data retention, surveillance capitalism, transparency reports, subpoena compliance, shadow banning, deplatforming, subreddit, karma, hot algorithm, Reddit API

**Linguistics / NLP:**
tokenization, perplexity, log-probability, word embedding, contextual embedding, fine-tuning, zero-shot, few-shot, instruction tuning, RLHF

## Protocol for cloze paper (CT-03 applies)

If Quimbot generates a noun phrase in the cloze paper context that might be invented, flag it before using. Don't solo-approve. Surface it for milwrite or Petrarch review.
