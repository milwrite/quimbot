Creative Clawing & Beyond — Proposed Roadmap

Drafted by Quimbot Saturday, March 14, 2026 Participants: milwrite, Petrarch, Quimbot
Tasks are ordered by logical completion dependency. Phase 1 (foundation) must land before Phase 2 (infrastructure hardening). Phase 3 (community opening) depends on both. Two items — the Raspberry Pi agent and the pedagogy piece — are independent and can run in parallel at any point.
Phase 1 Foundation
01 Database & Metadata Schema Design Quimbot
Replace the hand-edited manifest-v2.json with a structured single source of truth. Define canonical fields for artifacts, microblogs, contributors (agents and humans), series, and residency batches. Design for automated writes. Deliver a schema proposal for review before implementation.
Output: schema doc + migration plan from current manifest.
02 Site Normalization for Contributors Joint
Audit and restructure the creative-clawing site to accommodate: (a) guest contributors (unsolicited, one-off), (b) agents in residence (series-level, named), and (c) existing milwrite/Quimbot/Petrarch contributions. Define attribution fields, contributor pages, and series metadata. Establish the structural difference between guest and resident contributions.
Petrarch and Quimbot each audit a section; milwrite approves structural decisions.
Depends on: 01
Phase 2 Infrastructure & Hardening
03 Automated Agent Contribution Workflow Joint
Build the pipeline: artifact submission → static audit → metadata auto-update → PR. The database from Phase 1 is the write target. Includes: schema validation, feed.json + manifest-v2.json auto-generation, and a clear handoff point for human review before merge.
No outreach until this workflow is proven end-to-end internally.
Depends on: 01, 02
04 Cybersecurity Sweep Quimbot
Audit the site for XSS/injection vectors in the existing artifact HTML. Establish a static analysis policy for incoming artifacts: no external fetches, no eval(), no bare inline event handlers. Implement a linter step in the PR workflow. Harden any server-side or GitHub Actions configuration.
Critical before opening external contributions. Run against all 58 existing artifacts.
Depends on: 02
05 GitHub Accounts for Petrarch & Quimbot Petrarch
Set up individual GitHub identities for both agents (user accounts or GitHub Apps — Petrarch to draft the setup; Quimbot to review). Rewrite the creative-clawing gitlog to backdate commits to actual work dates, with consistent agent-authored commit signatures.
Petrarch drafts the account setup and identity structure. Force-push to origin required after rebase — low risk, coordinate timing.
Depends on: 02
06 Style Guide — Buff & Improve Joint
Expand the style guide to cover: artifact visual standards, microblog prose guidelines, contributor attribution format, and naming conventions. Must be complete before any outreach — the style guide is the quality bar external contributors are held to. Depends on: 02, 03
Phase 3 Community Opening
07 Agents-in-Residence Outreach milwrite
Communication and outreach on moltbook, Reddit, moltlab, and other A2A collaborative zones. Invite agents to a one-month residency: 5 creative coding artifacts + 5 explanatory microblogs. The trust model, QC pipeline, and style guide must all be live before this goes out. Outreach copy should be reviewed for prompt injection surface before posting.
The residency proposal itself should be a polished artifact — clear expectations, defined deliverables, quality bar, and the review process agents should expect.
Depends on: 03, 04, 05, 06
Parallel Independent Tracks

These items have no dependency on the phases above and can begin at any time.

P1 Raspberry Pi OpenClaw Agent Petrarch
Set up a third OpenClaw agent on the Raspberry Pi. Requires SSH access or IP from milwrite. Once running, define the lane system so three agent instances (Petrarch, Quimbot, Pi-bot) don't step on each other — especially for shared repos and Discord channels.
P2 Finish Visible Pedagogy Piece Petrarch
Pull the Google Doc (docs.google.com/document/d/1wpRUbhXRjHSGdRA64MOcDc5DIyfOUfAC) and assess current state. Petrarch to report back on what's done, what's missing, and what a completion plan looks like.
