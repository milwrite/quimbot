# Methodological Addendum: Generative AI as a Coding Scaffold

## How This Research Used Large Language Models Without Exposing Student Data

Throughout this dissertation, I employed large language models (LLMs) like Claude as coding scaffolds to support computational methods used for data collection, preparation, and analysis. I want to be precise about what this means and, more importantly, what it does not mean. At no point did an LLM process, read, or have access to the student data collected from Reddit; the language model never saw a single post, comment, username, or any fragment of the 273,702 records housed in my local databases. Its role was strictly limited to generating Python scripts and shell commands that I then reviewed, tested, and executed on my own machine against data stored locally.

There is a distinction to be made here between the AI tool that generates workable code and the development environment where I run scripts for research. When, for instance, I needed a script to query all comments from r/Baruch containing references to financial aid between March and June 2020, I would describe the task to the LLM in natural language and tie it to the specific database schema, table structure, or SQL filtering logic fit for the task at hand. The LLM would subsequently generate and return a Python script, which I would read, test, and modify before running it myself from the command line. In so many words, the agentic coding scaffold would generate the Python script from a description of the task, save it to my machine, and then allow me to run it locally in a separate and secure environment where accumulated findings live as SQLite datasets.

For a more specific example, I would often describe a computational task in plain language, something like "write a Python script that reconstructs discussion networks from a SQLite database of Reddit submissions and comments, linking commenters to original posters through parent ID relationships," and the LLM would produce a working draft. Sometimes the draft needed significant revision; sometimes it was close to what I needed. In either case, I was the one who executed it, examined the output, and decided what to do next.

Consider how this played out with the network reconstruction work I did to fill in gaps left by the discovery logic of my main Reddit scraper, where I built interaction networks across eight CUNY subreddit databases containing submissions and comments with fields like `author`, `parent_id`, `created_utc`, and `score`. The resulting script had to be able to parse Reddit's `t3_` prefix convention to link comments back to their parent submissions, filter out deleted accounts and bot activity, and output edge lists suitable for network analysis via NetworkX. When I outlined these requirements and submitted them to Claude Code, the LLM produced a script that saved to `scripts/analysis/network/`, which I would then run against each database independently. The script processed approximately 153,000 edges across the CUNY network, and the model that helped write it never encountered a single one of those edges or the usernames and post content that generated them.

Shell scripts made this workflow even more modular: when recurring operations like running the same analysis across all ten databases, for instance, I had the LLM draft shell wrappers that looped through database files and called the Python scripts sequentially. Scripts in `bin/` like `run_all_cuny_scrapers.sh` and `query-overview` follow this pattern and were driven by scripting methods such as the use of for-loops, path variables, a call to `sqlite3` or `python3`, and so on. In the process, the workflow creates multiple layers of separation between the tool and the data, and I want to walk through these concretely because I think concerns around AI and research ethics often assume a monolithic process where everything flows through a single system, and that is most certainly not how this process unfolded in my case.

All collected data lives in SQLite databases stored on my local machine. These are flat files on disk, not hosted on a server, not synced to a cloud service as part of the research pipeline, and not accessible via any API the LLM could reach. When I interact with a language model through its chat interface or through an agentic coding environment like Claude Code, the model has no mechanism to access my filesystem or read the contents of my databases. It generates text, specifically code, and returns it to me. The separation is architectural, not just procedural. Reddit data is already pseudonymous in that users choose their own handles and the platform does not require or display real names. That said, pseudonymity is not anonymity. Patterns of posting behavior, disclosed personal details, or unusual usernames could theoretically enable re-identification, which is precisely why I never fed post content or usernames into an LLM. The scripts I generated process this data locally. When I produced reports on bridge users, meaning individuals who posted across multiple CUNY subreddits and connected otherwise separate campus communities, the identification of those users, their posting patterns, and the network metrics characterizing their behavior were all computed by Python scripts running on my machine. The LLM helped me write the NetworkX code to calculate betweenness centrality and cross-subreddit participation ratios. It never saw who those bridge users were or what they wrote.

The project also includes IRB-protected interview data from four semi-structured interviews with CUNY students and staff. This material was handled with strict separation from the AI-assisted workflow entirely. Interview transcripts were never used as input to any LLM prompt, never referenced in code generation requests, and never processed by AI-generated scripts. The interview data exists in a separate directory with its own access protocols, and the analysis of that qualitative data was conducted manually, without computational scaffolding from language models.

The honest answer for why I used AI tools at all is that the computational scope of this project would have been impractical for a single researcher without some form of coding assistance. The LLM accelerated the scripting process in ways that let me focus on the interpretive and analytical work that constitutes the actual research. When I needed to write a temporal analysis script that bucketed 273,702 posts into monthly intervals and calculated activity ratios between pre-pandemic and post-pandemic periods, the LLM helped me get the datetime parsing and pandas groupby logic right faster than I would have managed alone. When I needed to identify early adopters, users whose first posts in CUNY subreddits predated 2015 and who remained active through the pandemic, the model helped me draft the SQL queries and Python filtering logic to isolate those accounts across multiple databases. In both cases, I described what I wanted in plain language, received code, reviewed it, and ran it locally. The alternative was not "do this research without AI." The alternative was either narrowing the scope to what I could code unassisted, which would have meant fewer subreddits, shorter timeframes, and less comprehensive network analysis, or spending additional years developing programming skills to a level where I could write these scripts from scratch. The LLM did not do the research. It helped me build the instruments I used to conduct it.

Because AI-generated code can contain errors, I treated every script the model produced as a draft requiring validation. I ran scripts on small subsets of data first, spot-checked outputs against manual SQL queries, and compared results across multiple analytical approaches to confirm consistency. When a network reconstruction script produced edge counts, I verified those counts against direct database queries, or when another script reported activity spikes across a given period of time, I cross-referenced the dates against known events like the March 2020 pandemic onset, Hurricane Sandy in 2012, and CUNY budget crises in 2016 to confirm the patterns were real rather than artifacts of code errors or confabulations courtesy of the coding agent.

Any computational research pipeline requires validation regardless of how the code was written. The difference here is one of utility, transparency, and replicability. In disclosing my use of AI as a coding scaffold for Python scripting and data collection and analysis, I hope readers will see those methods as a chance to critique the technology at the same time that they benefit from it, allowing them to evaluate AI's role in methodology beyond the confines of any one dissertation project. In this case, however, the interpretive claims and broad sweep of this project remain my own, accountable only to me, to my ethos as a researcher, and my dual identity as a teacher and a learner within the CUNY system.
