CUNY AI Lab · T(h)inkering Track

CALI Cohort Tool Prototypes

Five faculty-proposed tools for the t(h)inkering track. Each is scoped either as a dedicated web app or an agentic workspace configuration in the CUNY AI Lab Sandbox (Open WebUI), designed to be testable, reusable, and iterable beyond the initial chat interface.

Dedicated web app / standalone tool
Open WebUI workspace configuration
Tool 01
Research Writing Scaffold
K.C.
Open WebUI Workspace
What it does

A structured writing companion that guides researchers through iterative stages of academic writing: claim formation, evidence integration, paragraph-level revision, and citation scaffolding. The model responds to uploaded drafts or pasted text and returns stage-specific feedback rather than full rewrites, preserving the researcher's voice while surfacing structural weaknesses.

Sandbox configuration
Model preset
GPT-4o or Claude 3.5 Sonnet; strong instruction-following + long context for draft uploads
System prompt role
Academic writing coach; stage-gated feedback; never rewrites full passages
Knowledge collections
Discipline-specific citation guides (APA/MLA/Chicago), sample argument maps, genre exemplars
Tools enabled
file upload, web search (citation lookup), code interpreter (for quantitative claims)
Build path beyond chat UI
1
Export workspace preset as JSON; share URL with cohort members for one-click load
2
Companion web form: student pastes draft → backend calls Sandbox API → returns stage-tagged feedback as structured HTML
3
Google Docs add-on (Apps Script + Sandbox API) for in-document marginal feedback
4
Evaluation rubric embedded in system prompt; export feedback logs for iterative prompt tuning
Tool 02
Laboratory Workflow Tool
C.D.
Dedicated Web App
What it does

A protocol-aware lab assistant that parses standard operating procedures (SOPs) into step-by-step interactive checklists, flags safety checkpoints, and lets researchers annotate deviations in real time. Return to a paused protocol and resume with context intact. Can ingest uploaded PDFs of existing lab protocols.

Prototype architecture
Frontend
Single-page HTML + vanilla JS; step-navigator sidebar, annotation panel, progress state saved to localStorage
AI layer
OpenRouter API call on SOP upload → returns structured JSON of steps, safety flags, materials list
Persistence
Export completed session as annotated PDF; importable JSON for sharing across lab members
Evaluation loop
Log deviation annotations → surface patterns across sessions to flag protocol ambiguities
Build path
1
Static HTML prototype: manual SOP entry → stepwise checklist display
2
Add PDF upload + LLM parsing → auto-generate step list from uploaded protocol
3
Annotation layer + export → share with PI for review
4
Multi-user mode: shared session link, concurrent annotation by lab team
Tool 03
Deep Listening Tool
A.C.
Dedicated Web App
What it does

An audio-first qualitative research tool for interview-based scholarship. Upload or record spoken content; the tool transcribes, segments by speaker turn, and applies thematic tagging. Researchers surface recurring motifs, affective registers, and silences (the things that close reading of transcripts misses). Designed for ethnographers, oral historians, and STS researchers.

Prototype architecture
Transcription
Whisper (local via CLI or Groq API for speed) → timestamped JSON segments
Analysis layer
LLM pass over segments: thematic codes, affect detection, hesitation/silence markers, follow-up question suggestions
Frontend
Interactive transcript viewer: click segment → hear audio clip + see analysis annotations inline
Export
Coded transcript as DOCX/CSV; theme frequency chart; sharable annotated session link
Build path
1
Static viewer: paste existing transcript → apply manual thematic tags via UI
2
Add Whisper transcription endpoint → auto-populate transcript from audio upload
3
LLM annotation pass → surfaced codes appear as inline highlights
4
Researcher override mode: correct/reject LLM codes → retrain prompt with corrections for next session
Tool 04
Interdisciplinary Math Connections Tool
V.T.
Open WebUI Workspace
What it does

A concept-bridging assistant that helps educators and students discover how a given mathematical idea (topology, statistics, linear algebra, etc.) shows up in non-math disciplines: art history, urban planning, music theory, biology. Returns worked examples, visual analogies, and suggested readings. Intended to break the disciplinary isolation of math instruction at CUNY.

Sandbox configuration
Model preset
Claude 3.5 Sonnet; strong at analogy generation and cross-domain synthesis
System prompt role
Interdisciplinary math translator; always returns ≥2 non-math domains; includes a worked example per domain
Knowledge collections
Curated cross-disciplinary math texts (Ellenberg, Strogatz, Hofstadter), CUNY syllabus excerpts
Tools enabled
web search (find current research applications), image generation (visual analogies)
Build path beyond chat UI
1
Concept map web app: enter a math topic → interactive network graph of disciplinary connections
2
Each node in the graph links to a generated "connection card" with example + reading
3
Teacher mode: export connection cards as printable handouts or slide deck
4
Community layer: submit your own connection examples → build a CUNY-specific cross-disciplinary corpus
Tool 05
Systems and Assets Mapping Tool
M.M.C.
Dedicated Web App
What it does

A visual mapping environment for community-based and participatory action research. Users add actors, institutions, resources, and relationships to a live canvas; the AI layer surfaces connection gaps, power asymmetries, and underutilized assets based on the mapped structure. Outputs shareable SVG maps and structured data for grant applications and program planning.

Prototype architecture
Canvas layer
Force-directed graph (D3.js or Cytoscape.js); drag-drop node creation; edge type labels (resource, power, information)
AI analysis layer
Submit graph JSON → LLM returns: isolated nodes, bridging opportunities, missing actor types, narrative summary
Input modes
Manual node entry; CSV import of stakeholder list; text description → AI-generated draft map
Export
SVG/PNG map; JSON for re-import; narrative analysis as DOCX; embed-ready iframe
Build path
1
Static canvas prototype: manual node + edge creation, visual grouping by category
2
Add CSV import → auto-populate nodes from stakeholder spreadsheet
3
LLM analysis panel: submit current map → receive gap analysis and narrative summary
4
Collaborative mode: shared map URL, real-time co-editing by community members
Tool 06
Music Imposition and Decomposition Tool
Deep Listening Extension
Dedicated Web App
What it does

A browser-based tool for separating and layering musical components to support pedagogical and analytical deep listening. Upload a recording; the tool decomposes it into stems (vocals, bass, drums, harmony, other) using source separation, then lets the listener isolate, mute, solo, or re-layer any combination. Works in both directions: decompose existing recordings to hear what was previously obscured, or impose layers together to explore how musical elements interact. Designed for music educators, ethnomusicologists, and researchers working with sound as primary data.

Prototype architecture
Decomposition engine
Demucs (Meta, open source) or Spleeter for stem separation; runs server-side or via Hugging Face Inference API
Playback layer
Web Audio API: per-stem gain nodes, mute/solo toggles, synchronized playback cursor across all stems
Imposition mode
Drag stems from multiple sources onto a shared timeline; adjust relative volume and timing offset; export as mixed WAV
Annotation layer
Click any moment on any stem to add a text note; tag by analytical category (timbre, rhythm, harmony, texture); export annotated score as PDF or CSV
Build path
1
Static prototype: upload pre-separated stems manually, build the multi-track mute/solo player with Web Audio API
2
Add separation endpoint: audio upload triggers Demucs via HF API, returns 4 or 5 stem files
3
Annotation panel: timestamp-linked notes per stem, exportable as structured data
4
Imposition workspace: drag stems across sessions, build composite arrangements for comparative analysis or teaching demonstrations
