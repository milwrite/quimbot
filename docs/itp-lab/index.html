Prompt Anatomy

Structure prompts for creative code

  • Set role (creative coding generator/collaborator)
  • Enforce constraints (vanilla JS; no libraries; Canvas; viewport)
  • Specify aesthetics (monochrome; early computer art; geometric)
  • Define technical frame (requestAnimationFrame; interaction)
  • Demand output format (single HTML; no explanation)
ROLE: You are a creative coder...
CONSTRAINTS: vanilla JS only; Canvas; full viewport; no external libs
AESTHETIC: monochrome; geometric; early computer art
TECH: requestAnimationFrame; mouse interaction
FORMAT: return single HTML only (no markdown/explanation)

Activity 1 (10m)

One prompt, one artifact

  • Paste Activity 1 system prompt in Sandbox
  • Set temperature 0.9; max tokens 4096
  • Write one natural-language prompt; do not iterate
  • Copy HTML → save → open in browser

Artifact: example “starfield” output; move pointer to shift parallax.

Activity 2 (15m)

Iterate and refine

  • Switch to Activity 2 system prompt (adds explanation + suggestions)
  • Iterate ≥3 rounds: generate → test → request changes → regenerate
  • Connect workflow: generate/test/describe/regenerate
Say: “Keep the artifact, but add trails / change palette / add speed control UI.” Then re-test each full HTML output.

Activity 3 (15m)

Write the rules, not the request

  • Author your own constraint-based system prompt
  • Set temperature 0.5–1.5; max tokens 1k–3k
  • Run “Generate” 3×; select favorite output (curation)
RULES (example):
1) Single HTML with inline CSS+JS
2) Canvas only; no libraries
3) Grid ≥ 100 elements
4) Vary property by distance from center
5) Exactly one randomness source
6) Only 3 colors
7) Fill viewport
8) Animate with requestAnimationFrame
Return HTML only

Reflection

Map activities to triangle

  • Activity 1: high Interpretation; moderate Randomness; low Rules
  • Activity 2: high Interpretation; moderate Rules; moderate Randomness
  • Activity 3: high Rules; variable Randomness; curation at end

Takeaways
