Inference as Coding Scaffold

Using language models to reduce the gap between intent and implementation

A model doesn't write your code for you. It closes the distance between what you want and what the machine can execute ‚Äî faster than a tutorial, slower than a colleague who already knows the answer.

The scaffold collapses as you learn. What starts as "write me a function that does X" becomes "why does this function fail when the input is empty" ‚Äî and eventually you're reading the output critically rather than accepting it.

Problematics of Agentic Coding

When the model acts, not just responds

Agentic coding tools don't just suggest ‚Äî they write files, run commands, and make decisions in sequence. The loop runs faster than you can read it.

This is useful and dangerous in the same breath. You get to your goal faster. You also lose track of what the machine did to get there. Understanding what you are delegating, and when to take back control, is the central skill.

‚ö°
Speed
Agentic loops produce results in seconds. The pace can outrun your ability to verify what's happening.
ü™§
Attribution loss
If something breaks, do you know why? Agent-generated code can produce working results you cannot explain or debug.
üéõÔ∏è
Deliberate checkpoints
Interrupt, read, commit to version control, then continue. The checkpoint is where your judgment lives.

Versioning & Project Management with Git

The history of your work as a navigable record

Git tracks every change you make to a project. It is not a backup ‚Äî it is a timeline. You can move forward, backward, and sideways through your own history.

For research and writing projects, this means you never lose a working version of anything. For code, it means you can try something and undo it completely if it fails.

# start tracking a project $ git init # save the current state $ git add . $ git commit -m "first working version" # review history $ git log --oneline # go back to a previous version $ git checkout a3f91c2

Social Coding + Collaboration with GitHub

Git on the internet, with an audience

GitHub puts your Git history online. Your repository becomes a URL ‚Äî something you can share, link in a CV, and collaborate on with anyone who has a browser.

For this workshop, GitHub is also where your built artifacts live. GitHub Pages turns any repository into a static website. Ship a single HTML file and it is immediately accessible to anyone in the world.

Command Line Basics

Talking to your computer without a GUI

The command line is a text interface to the operating system. It feels unfamiliar, but it follows simple rules: every command is a verb, followed by options, followed by a target.

You do not need to memorize commands. You need to know what category of thing you are trying to do, and how to look up the right command for it.

# where am I? $ pwd # what's in this folder? $ ls -la # move into a folder $ cd my-project # create a new file $ touch index.html # open current folder in VS Code $ code .
