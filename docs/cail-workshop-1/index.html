Concept

Inference as Coding Scaffold

Prompts as prototypes — Karpathy, Feb 2025

Inference closes the gap between intention and implementation. You describe what you want; the model writes the syntax. The code becomes legible because you steered it.

Three modes, increasing control: one-shot generation → iterative refinement → constraint-based authorship. Each round, you move from interpreter to architect.

Spectrum of control
One-shot
Generate once. Observe what the model assumes.
Iterative
Generate → test → describe → request one change → repeat.
Constraint-based
Write the rules, not the request. You are the architect.
Concept

Problematics of Agentic Coding

No-review vs deliberate iteration

"Vibe coding" (Karpathy) means accepting output without reading it. It moves fast. It also accumulates technical debt invisibly — the model's assumptions become load-bearing.

Decomposing tasks into smaller subtasks increases the number of decision points where you can intervene, evaluate, and redirect. More checkpoints means more deliberate output. — Xu et al., 2024

The diagnostic question
No-review mode
Accept output. Run it. Move on. Fast but opaque.
Deliberate mode
Read. Test. Describe the gap. Request one change. Repeat.
Ask: what did the model assume? That's the beginning of oversight.
